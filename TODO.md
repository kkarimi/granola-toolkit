# V1 Candidate TODO

Previous completed roadmap: `V1-FOUNDATION-TODO.md`.

North star: make `granola-toolkit` the programmable post-meeting layer on top of Granola, with BYOA note generation, durable review and approval, reliable transcript reprocessing, and provider choice instead of one built-in AI path.

## V1 Candidate Guardrails

- Stay in one TypeScript repo and one npm package for now.
- Keep sync, eventing, agent execution, review, and exports as separate layers.
- Treat every agent run as a durable job with replayable inputs, outputs, and provenance.
- Keep provider support abstraction-first, with OpenRouter plus OpenAI/Codex as first-party targets.
- Support per-folder, per-tag, per-meeting-pattern, and per-recurring-meeting harnesses.
- Let users keep prompts and meeting playbooks in local files such as `AGENT.md`-style instructions rather than only inline config.
- Keep human approval optional but first-class for writes, enrichments, and external actions.
