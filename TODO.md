# V1 TODO

North star: evolve `granola-toolkit` from a strong local workbench into a durable Granola automation platform with explicit sync, programmable actions, API-key-first auth, a proper web app, and polished public releases.

## V1 Guardrails

- Stay in one TypeScript repo and one npm package for now.
- Keep sync, local state, eventing, automation rules, and actions as separate layers.
- Default to polling and local persistence until Granola ships webhooks.
- Prefer Granola API keys as the primary auth path; keep stored sessions and `supabase.json` as fallbacks.
- Keep CLI, TUI, server, and web on the same shared app contracts.

Status note:
As of 4 April 2026, Granola developer API keys are visible in the desktop settings UI, and Granola’s public docs currently position API keys as the intended path for scripts and automations. Sync should therefore treat polling as the current backbone, with webhooks considered later if Granola adds them.
