# Beta TODO

North star: evolve `granola-toolkit` from a strong export CLI into a modular Granola workbench with three faces on one shared core:

- batch CLI commands
- a full-screen terminal UI built with [`pi-tui`](https://github.com/badlogic/pi-mono/tree/main/packages/tui)
- a browser client launched by `granola web`, inspired by [`opencode`](https://github.com/anomalyco/opencode)

## Beta Guardrails

- Stay in one TypeScript repo and one npm package for now.
- Keep transport, domain, and client boundaries clean so a future workspace split is optional, not required now.
- Treat the current CLI as the first client of the shared app core, not a side path to be rewritten later.
- Default to local-first and secure-by-default server behaviour.
