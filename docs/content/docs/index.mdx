Granola Toolkit started as an exporter and is now a local workbench for Granola meetings.

The next v1 foundation is explicit local sync. `granola sync` now refreshes the shared meeting
index and records structured changes instead of leaving refresh entirely to opportunistic command
paths.

It has three user-facing surfaces on one shared TypeScript core:

- batch CLI commands
- a browser workspace from `granola web`
- a terminal workspace from `granola tui`

## Core Ideas

### Shared app core

CLI commands, the browser workspace, and the TUI all talk to the same application core for auth,
meeting discovery, folders, transcripts, notes, and export jobs.

### Local-first server

`granola serve` is the seam that lets different clients share one live stateful app instead of
repeating the same work in separate processes.

### Folders are first-class

Folders are not bolted on as a filter. They are part of the shared domain model and are available
through CLI commands, the browser, the TUI, and export flows.
