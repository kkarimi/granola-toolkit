## Install

```bash
npm install -g granola-toolkit
granola --help
```

If you do not want a global install:

```bash
npx granola-toolkit --help
```

The package exposes both `granola` and `granola-toolkit` as executable names.

## First Run

Store a Granola Personal API key once:

```bash
granola auth login --api-key grn_...
granola auth status
```

If you prefer to reuse the desktop app session, `granola auth login` still imports it from
`supabase.json`.

Then verify that folders and meetings resolve cleanly:

```bash
granola sync
granola sync --watch
granola automation rules
granola folder list
granola meeting list --limit 10
granola meeting view 1234abcd
```

## Typical Starting Commands

```bash
granola sync
granola notes
granola notes --folder Team
granola transcripts
granola web
granola tui
```

## When to Use Which Surface

- Use the CLI when you want scripts, exports, or stable machine-readable output.
- Use `granola web` when you want quick navigation, tabs, and export actions in a browser.
- Use `granola tui` when you want the same workspace model with a keyboard-first terminal UI.
