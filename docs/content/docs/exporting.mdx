---
title: Exporting
description: Keep Granola notes and transcripts in durable local files you control, with stable incremental export state.
---

Exports are still a core part of the toolkit, but the point is no longer “dump markdown once”.
The point is to keep a local copy of your notes and transcripts in formats you can search, version,
review, and feed into other tools.

## Notes

`granola notes` exports Granola's generated meeting notes, not the raw transcript.

The flow is:

1. Resolve credentials, preferring a stored API key, then stored session, then `supabase.json`.
2. Call the appropriate Granola notes API for the active auth mode.
3. Normalise each document into a structured note export.
4. Choose the best available note content.
5. Render Markdown, JSON, YAML, or raw JSON.
6. Write one file per document into the output directory.

Content is chosen in this order:

1. `notes`
2. `last_viewed_panel.content`
3. `last_viewed_panel.original_content`
4. raw `content`

Examples:

```bash
granola notes
granola notes --folder Team
granola notes --format json --output ./notes-json
```

## Transcripts

`granola transcripts` exports locally cached transcript segments.

The flow is:

1. Read Granola's cache JSON from disk.
2. Parse the cache payload.
3. Normalise transcript data into a structured export per document.
4. Match segments to documents by document id.
5. Render text, JSON, YAML, or raw JSON.
6. Write one file per document into the output directory.

Examples:

```bash
granola transcripts
granola transcripts --folder Team
granola transcripts --format yaml --output ./transcripts-yaml
```

## Folder-Scoped Exports

When you pass `--folder <id|name>`, the export is filtered to that folder and, by default, written
into a stable per-folder subdirectory under the output root.

Examples:

```bash
granola notes --folder Team
granola transcripts --folder Team
```

## Export Jobs

Exports are tracked as jobs with persistent history, scope metadata, status, progress counters, and
rerun support, so the browser and future automation flows can treat exports as part of the shared
local runtime instead of a fire-and-forget shell command.

```bash
granola exports list
granola exports rerun notes-1234abcd
```

## Incremental Writes

The exporter keeps a hidden state file in the output directory to track:

- document id to filename
- content hash
- source timestamp
- last export time

That state is used to keep filenames stable, skip unchanged renders, migrate format changes, and
delete stale files.
